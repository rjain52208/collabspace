# Contributing to CollabSpace

Thank you for your interest in contributing!

## Development Setup

1. Fork the repository
2. Clone your fork
3. Follow the QUICKSTART.md guide
4. Create a new branch for your feature
5. Make your changes
6. Test thoroughly
7. Submit a pull request

## Code Style

### Python (Backend)
- Follow PEP 8
- Use Django best practices
- Add docstrings to functions

### JavaScript (Frontend)
- Use functional React components
- Follow React hooks best practices
- Add comments for complex logic

## Testing

Before submitting PR:
- Test real-time collaboration with 2+ users
- Verify WebSocket connections are stable
- Check console for errors
- Test on Chrome and Safari

## Pull Request Process

1. Update README.md if needed
2. Describe your changes clearly
3. Link any related issues
4. Wait for review

## Questions?

Open an issue on GitHub for any questions or suggestions.
